   │  └───────────┘   │

            │

     ├─ WeChatAdapter   → 微信 API
     └─ 未来Adapter      → 新平台 API
```

### 核心设计要点

| 要点 | 说明 |
|------|------|
| **ChannelMessage** | 平台无关的消息模型，携带 `reply_callback` 闭包，队列消费者无需知道消息来自哪个平台 |
| **ChannelAdapter** | Protocol 类，新平台只需实现 `send_reply(user_id, text)` |
| **TaskQueue 单例** | 飞书和微信共享一个队列实例，统一管控并发、优先级、状态 |
| **完成即移除** | worker 执行完毕后自动从活跃集合删除，保留最近完成记录供 `/status` 查询 |
| **接入新平台** | 只需：① 实现 XxxAdapter ② 收到消息后构造 ChannelMessage ③ 调用 `queue.enqueue()` |
